# On solving equations (for some variables)
- Мысли как о графе атомарных операций который мы преобразуем в другой граф по опредеённым правилам
чтобы найти недостающие объекты графа.
Solving equation/inequality/etc means finding a set of objects (values) that make a function evaluate to true. 
This process can also be seen as a function: takes a function as input and returns a set of values that make that function evaluate to true.
Algebraic manipulations are just using equivalence relations (properties like commutativity, associativity) to manipulate the order and types of operations used. Those algebraic operations are replacement operations, they replace a bunch of functions composed in some way with a bunch of functions composed in another way, hoping that a new way gives us an opportunity to just calculate all possible values of x (argument), that we were not able to calculate before.
When we turn sum(2, 5) into sum(2, 5) = identity(x), we just create a function that return True/False and takes one argument. Now the question might be: what set of arguments make this function return True?
Example 1:
equal(successor(x), 4) - find a set of values for x, that this function maps to 1 (True)
equal(x, predecessor(4)) - replacement step
equal(x, 3) - replacement step
x = {3} - found a set
Example 2 (finding a value of a function that is not defined in terms of sequence of simpler operations, but in terms of equality (a^log(a, b) = b). So, the log function mast be such that the former function returns true: 
log(3, 9) - do not know the answer
log(3, 9) = x or equal(log(3, 9), x) - Find a log means find a create a new function and find an argument x that returns True.
exp(3, x) = 9 - manipulate the expression using an equivalence relation. This new function return True for exactly the same set of inputs
exp(3, x) = exp(3, 2) - equivalence relation from previously calculated exp(3, 2) = 9.
Now we see that only 2 makes this function return True.
For some particular function, there is a defined way how to find it's value from it's arguments. But, there is no described way of how to find a set of arguments that make a function evaluate to some particular value (or a set of values).
We describe such rules in logic.
Examples of such descriptions:
equal(successor(a), b) -> equal(a, predecessor(b))
equal(and(a, 1), 1) -> equal(a, 1)
equal(plus(a, b), c) -> equal(a, minus(c, b))
So, basically, changing the order and types of functions applied to arguments, to get an answer what those arguments might be.

    - __Нулевая толщина__ - толщина уровня одной точки реального постранства, когда мы не можем сделать
    даже 1 шаг по реальному пространству (в направлении нулевой толщины).
- __Евклидово пространство__ - подмножество точек реального пространства, помеченное определёным
систематическим способом: ьак, чтобы число шагов реального пространства было равно sqrt(2) меду точками
которые отличаются на 1 единицу в обеих направлениях, как (0, 0) и (1, 1).
- __Примерный процесс создания Евклидового пространства__ - взяли произвольную точку, провели через неё
3 перпендикулярные друг другу прямые, выбрали минимальную длины реального пространства, задали точкам
лейблы векторами - вектор-лейбл для точки А определяется найдя точки на прямых, близжайшие к точке А и
далее определив расстояния этих точек от ориджин точки.
- __Евклидово пространство имеет направдение__, так что измерять число реального пространства с помощью
подсчёта евклидовых точек будет создавать погрешность, и чем меньше совпадает направдение фигуры
с направлением осей, тем больше будет погрешность.
Даже если делать поправку на то что диагональный шаг стоит sqrt(2), всё равно будет погрешность.
- __Нас интересуют шаги реального пространства__ - шаги евклидового пространства это абсолютно
нерелевантная вещь.
